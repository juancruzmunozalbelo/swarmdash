# Contributing to SwarmDash

Thank you for your interest in contributing to SwarmDash! 🎉

## How to Contribute

### Reporting Bugs

1. Open an [issue](https://github.com/juancruzmunozalbelo/swarmdash/issues)
2. Include steps to reproduce, expected vs actual behavior
3. Add screenshots if applicable

### Suggesting Features

Open an issue with the `enhancement` label describing the feature and its use case.

### Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes and commit: `git commit -m 'feat: add my feature'`
4. Push to your fork: `git push origin feature/my-feature`
5. Open a Pull Request

### Development Setup

```bash
git clone https://github.com/juancruzmunozalbelo/swarmdash.git
cd swarmdash
# Open index.html directly or serve with any static server:
python3 -m http.server 8888
```

### Commit Convention

We follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — new features
- `fix:` — bug fixes
- `docs:` — documentation changes
- `refactor:` — code refactoring

### Code Style

- Vanilla HTML/CSS/JS — no build tools needed
- Keep code modular and well-commented
- Follow existing naming conventions

## Code of Conduct

Be respectful, inclusive, and constructive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/).
